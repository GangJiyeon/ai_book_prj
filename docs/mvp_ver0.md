## 1. 서비스 한 줄 정의

이 서비스는 **문장 기반 소셜 리딩 플랫폼**이다.
기존 전자책 플랫폼처럼 책을 먼저 선택해서 읽는 구조가 아니라, **문장을 먼저 발견하고 그 문장을 중심으로 댓글과 토론이 형성되며, 이후 책 전체로 관심이 확장되는 구조**를 가진다.

핵심은 책 자체보다 **문장(sentence)** 이다.
사용자는 인상적인 문장을 보고, 그 문장에 대한 다른 사람들의 생각을 읽고, 직접 댓글을 남기며, 필요하면 책 전체로 넘어간다.

즉 이 서비스의 핵심 흐름은 아래와 같다.

**문장 발견 → 댓글/토론 → 공감/저장 → 책 관심 형성 → 독서 확장**

---

## 2. MVP의 핵심 목표

### 핵심 가설

* 사용자들은 **문장을 중심으로 다른 사람들과 생각을 나누는 경험**에 흥미를 느낀다.
* 사용자들은 책 전체보다 먼저 **문장 단위 콘텐츠**를 소비하는 방식에도 익숙하게 반응할 수 있다.
* 사용자들은 직접 문장을 등록하고, 댓글을 남기고, 저장하는 식으로 **UGC 기반 참여**를 할 수 있다.

### MVP 목표

* 사람들이 실제로 문장 피드를 소비하는지
* 특정 문장에 댓글을 남기는지
* 댓글이 실제 토론처럼 이어지는지
* 사용자가 직접 문장을 업로드하는지
* 문장 기반 경험이 책 관심이나 독서 행동으로 이어지는지

## 4. MVP 사용자 시나리오

### 시나리오 A: 탐색형 사용자 ✅

1. 사용자가 앱에 들어온다.
2. 메인 피드(`/`)에서 문장 카드를 스크롤한다.
3. 문장 카드에서 책 제목/저자/댓글 수/좋아요 수를 확인한다.
4. 문장 클릭 → 상세 화면(`/sentences/[id]`)에서 댓글을 본다.
5. 공감되는 댓글에 좋아요를 누르거나 직접 댓글을 남긴다.
6. 피드에서 저장 버튼으로 문장을 저장한다.

### 시나리오 B: 생산형 사용자 ✅

1. 사용자가 자신이 읽은 책에서 인상적인 문장을 발견한다.
2. 네비게이션 `Share` 버튼 → `/sentences/new`에서 문장을 업로드한다.
3. 책 제목, 저자, 문장(최대 200자)을 입력한다.
4. 제출 후 피드로 복귀.
5. ⬜ 다른 사용자의 반응 알림 기능은 미구현.

### 시나리오 C: 독서 확장형 사용자 🔧

1. 사용자가 피드에서 문장을 발견한다.
2. 댓글과 토론을 보며 흥미를 느낀다.
3. ⬜ 문장 상세에서 책 정보 페이지로의 연결 링크 미구현.
4. 🔧 `app/read/`에서 독서 가능하나, 리더에서 피드로 문장 업로드하는 흐름 미연결.

## 5. MVP 핵심 화면 구성

### 5-1. 메인 화면: 문장 피드 ✅ `app/page.tsx`

#### 구현된 것
* 세로 스크롤 카드형 피드
* 문장 내용(serif 폰트, 큰 텍스트)
* 책 제목 / 저자 (보조 정보)
* 좋아요 수 / 댓글 수 / 저장 버튼
* 정렬 탭: Trending / Newest / Most commented
* 문장 클릭 → 상세 화면 진입

#### 미구현/보완 필요
* ⬜ 저장 상태가 새로고침 시 초기화됨 (localStorage 또는 서버 연동 필요)
* ⬜ 좋아요 상태도 동일하게 미영속

---

### 5-2. 문장 상세/댓글 화면 ✅ `app/sentences/[id]/page.tsx`

#### 구현된 것
* 문장 전체 표시 (serif 폰트)
* 책 제목 / 저자
* 좋아요 수 / 저장 수 + 토글
* 댓글 목록 (사용자명, 시간, 댓글 좋아요)
* 댓글 작성창 (엔터/버튼 제출)
* 작성한 댓글 즉시 표시 (로컬 state)

#### 미구현/보완 필요
* ⬜ 댓글 정렬 (시간순/인기순) 탭
* ⬜ 댓글 영속성 (새로고침 시 초기화)
* ⬜ 문장 상세에서 책 전체로 넘어가는 링크

---

### 5-3. 문장 등록 화면 ✅ `app/sentences/new/page.tsx`

#### 구현된 것
* 책 제목 / 저자 / 문장 내용 입력
* 글자 수 제한: **200자** (남은 글자 실시간 표시, 20자 미만 시 빨간색)
* 제출 후 1.5초 → 피드로 자동 이동
* 간단한 단계 구성 (스크롤 없는 한 화면)

#### 미구현/보완 필요
* ⬜ 제출한 문장이 실제 피드에 반영되지 않음 (mock 데이터 구조)
* ⬜ 선택: 책 표지 이미지 업로드
* ⬜ 선택: 출처 페이지 번호

---

### 5-4. 저장한 문장 화면 🔧 `app/saved/page.tsx`

#### 구현된 것
* 저장한 문장 리스트 (카드형)
* 각 문장의 책 제목/저자 표시
* 댓글 보러 가기 링크 (`/sentences/[id]`)
* 저장 해제 버튼 UI

#### 미구현/보완 필요
* ⬜ 피드에서 저장한 문장이 이 화면에 실제로 반영되지 않음 (상태 공유 미연결)
  → 현재는 mock 데이터 상위 3개가 고정 표시됨

---

### 5-5. 퍼블릭 도메인 리더 화면 🔧 `app/read/[id]/[chapter]/page.tsx`

#### 구현된 것
* 챕터 텍스트 표시
* 단락 클릭 → 댓글 패널 (데스크톱) / 바텀시트 (모바일)
* 단락별 댓글 작성 및 조회

#### 미구현/보완 필요
* ⬜ 읽다가 특정 문장을 피드에 업로드하는 흐름 연결
* ⬜ 피드 문장 카드 → 책 상세 → 리더로 이어지는 full flow

## 9. MVP 성공 판단 기준 (변동 없음)

### 특히 중요한 지표

* **문장을 본 사용자 중 댓글 화면까지 가는 비율**
* **댓글을 본 사용자 중 직접 댓글을 남기는 비율**
* **사용자가 직접 문장을 업로드하는 비율**
